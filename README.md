# ListingLaunch

An AI-powered real estate listing package generator. Paste in a property's details and photos, and get a complete marketing kit in seconds — email campaign, social media captions, PDF flyer, and a short-form video — all written in a consistent brand voice.

---

## Features

| Output | Description |
|--------|-------------|
| **Email** | HTML email with specs table and CTA, ready to copy into any email client |
| **Social Captions** | Platform-tuned posts for Instagram, LinkedIn, and Facebook with phone-frame preview |
| **PDF Flyer** | Downloadable print-ready flyer with photos, headline, specs, and contact footer |
| **Reels Video** | 30-second 9:16 animated video with Ken Burns photo effects, slide text, and background music |

All content is generated by Claude using a customizable system prompt that defines the agent's tone, vocabulary, and formatting rules.

---

## Tech Stack

- **[Next.js](https://nextjs.org/)** (App Router) — full-stack React framework
- **[Anthropic SDK](https://github.com/anthropics/anthropic-sdk-typescript)** — Claude API integration (claude-sonnet-4-6)
- **[Remotion](https://www.remotion.dev/)** — programmatic video composition and player
- **[jsPDF](https://github.com/parallax/jsPDF)** + **[html2canvas](https://html2canvas.hertzen.com/)** — client-side PDF generation
- **[Tailwind CSS v4](https://tailwindcss.com/)** — styling
- **[Zod](https://zod.dev/)** — runtime schema validation
- **TypeScript** throughout

## Getting Started

### Prerequisites

- Node.js 18+
- An [Anthropic API key](https://console.anthropic.com/)

### Installation

```bash
git clone <repo-url>
cd listinglaunch
npm install
```

### Environment Variables

Create a `.env.local` file in the project root:

```env
ANTHROPIC_API_KEY=sk-ant-...
```

That is the only required variable.

### Development Server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

### Production Build

```bash
npm run build
npm start
```

## How It Works

1. **User fills out the property form** — address, price, beds/baths, square footage, highlights, property type, and up to 3 photos (uploaded as base64).

2. **Form submits to `/api/generate`** — the server-side route constructs a prompt from the property data, attaches the photos as vision inputs, and calls the Claude API.

3. **Claude returns a `GeneratedPackage`** — a JSON object with five fields:

   ```ts
   {
     emailHtml: string;        // Full HTML email body
     instagramCaption: string; // IG caption with hashtags
     linkedinCaption: string;  // Professional-tone post
     facebookCaption: string;  // Community-tone post
     videoScript: string;      // 4 slide texts separated by " | "
   }
   ```

4. **OutputPanel renders each output** in its dedicated tab component.

## Customizing the AI Voice

Edit `src/lib/agent-voice.ts` to change the system prompt. The file controls:

- **Tone and personality** — how formal or conversational the copy sounds
- **Forbidden phrases** — words and patterns to never use (e.g. "stunning," "breathtaking")
- **Platform-specific rules** — character limits, hashtag conventions, emoji use per platform
- **Video script format** — how slides are structured and delimited

This is the most impactful file to edit when adapting the tool for a different agent or brand voice.

## Video Export

The Reels video uses [Remotion](https://www.remotion.dev/) for in-browser rendering. Because server-side rendering with `@remotion/renderer` requires Chromium (not practical for serverless deployments), video export uses the browser's **MediaRecorder API**:

1. Click "Download Video" in the Reels tab.
2. A 3-second countdown starts, then the video plays from the beginning.
3. The screen-capture recording stops automatically when playback ends.
4. The `.webm` file is saved to your downloads folder.

To convert to `.mp4`, run:

```bash
ffmpeg -i listing-reel.webm -c:v libx264 -c:a aac listing-reel.mp4
```
